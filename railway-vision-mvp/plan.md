# VisionHub 重构计划

基于 2026-03-03 对现有文档、代码和运行中页面 `https://localhost:8443` 的快速体验审计。

## 当前进度

- PR1 已完成：信息架构、首屏骨架、导航收敛
- PR2 已完成：设计 token、字体、按钮/输入/卡片/状态/表格/Toast/抽屉统一
- PR3 已完成：空状态、加载态、错误态、首次使用引导
- PR4 已完成：键盘可达性、抽屉焦点管理、对比度与响应式细修

## A. 快速体验审计

### A1. 核心任务最短路径

核心任务定义：客户用户上传资产并拿到一次可追溯的推理结果。

最短路径控制在 3 步：

1. 在“资产”上传图片或视频，获得 `asset_id`
2. 在“任务”选择流水线并创建任务
3. 在“结果”查看结构化结果和截图

管理员最短路径：

1. 在“模型”审批或发布模型
2. 在“更多 > 流水线”发布可调用 Pipeline
3. 在“结果”或“更多 > 审计”验证交付情况

供应商最短路径：

1. 在“模型”提交模型包
2. 等待平台审批
3. 在模型列表查看状态变化

### A2. 当前页面阻碍上手的 Top 10 问题

1. 导航入口过多。首页之外直接暴露 7 个页面，首次登录不知道先去哪。
2. 首页信息重复。指标、业务线、常用入口、命令条都在做“导航”这件事。
3. 首屏没有明确主路径。用户看不到“先做什么，再做什么，最后得到什么”。
4. 页面标题像系统说明，不像任务导向。大量“查询任务状态”“查看结果”是结果描述，不是下一步动作。
5. 顶部命令条按钮过多。多个页面同时出现 4 到 5 个动作，主次不清。
6. JSON 表单暴露太早。任务页和流水线页一进入就看到大段 JSON，认知负担过大。
7. 页面切分过细。`模型 / 流水线 / 任务创建 / 任务监控 / 结果 / 审计` 对新用户来说是实现视角，不是业务视角。
8. 空状态太弱。很多区域只有“点击刷新加载…”，没有告诉用户下一步该做什么。
9. 登录页噪音偏多。主题切换、快速角色、三列能力说明同时出现，干扰主动作“登录”。
10. 视觉语言不够收敛。当前有较强渐变、较多阴影、较大的区块密度，与“克制柔和”的目标不一致。

### A3. 删减清单

必须删：

- 首页“业务线”和“常用入口”双重导航并存
- 页面命令条中的多余动作堆叠
- 登录页非必要的能力宣传列表
- 首屏大段解释性副文案

建议删：

- 默认展示的全量 JSON 示例
- 结果页和审计页重复的“查询说明”
- 多处“定位 ID / 刷新列表”并列出现

可保留：

- 当前页面与接口结构
- 模型、流水线、资产、任务、结果、审计现有业务逻辑
- 抽屉详情、Toast、空状态、分页等基础交互
- 通过 hash 切页的当前兼容方式

## B. 信息架构重构

### B1. 新导航

一级入口最多 5 个：

1. 开始
2. 模型
3. 资产
4. 执行
5. 结果

放入“更多”的页面：

- 流水线
- 任务监控
- 审计

兼容策略：

- 保留原页面 ID 与 hash 路由：`#/dashboard`、`#/models`、`#/pipelines`、`#/assets`、`#/tasks`、`#/task-monitor`、`#/results`、`#/audit`
- 只调整导航分组、页面标题和首屏骨架，不改后端接口，不改页面数据契约

### B2. 页面层级定义

统一层级：

- 标题区：页面名称 + 一句结果导向说明
- 主行动区：本页唯一主按钮 + 关键输入
- 辅助区：次要动作、筛选、状态说明
- 结果区：列表、状态卡、详情抽屉
- 帮助区：仅保留简短空状态和下一步建议

### B3. 页面线框

#### 1. 开始

| 区域 | 内容 |
| --- | --- |
| 标题区 | 角色欢迎语 + “3 步完成一次交付” |
| 主行动区 | 三个步骤卡：准备能力 / 上传资产 / 执行并看结果 |
| 辅助区 | 今日常用入口，最多 3 个 |
| 结果区 | 最近任务 |
| 帮助区 | 无数据时提示“先上传资产”或“先提交模型” |

#### 2. 模型

| 区域 | 内容 |
| --- | --- |
| 标题区 | 模型管理 |
| 主行动区 | 提交模型包 或 审批发布 |
| 辅助区 | 筛选、状态切换、进入更多 > 流水线 |
| 结果区 | 模型列表 |
| 帮助区 | 无模型时提示“先提交模型包” |

#### 3. 资产

| 区域 | 内容 |
| --- | --- |
| 标题区 | 上传资产 |

| 辅助区 | 默认值说明 |
| 结果区 | 上传成功卡 + `asset_id` |

#### 4. 执行

| 区域 | 内容 |
| --- | --- |
| 标题区 | 创建任务 |
| 主行动区 | 选择流水线、输入资产 ID、创建任务 |
| 辅助区 | 执行方式切换、上下文和高级选项 |
| 结果区 | 创建结果卡 + 跳转结果 |
| 帮助区 | 无流水线时提示去“更多 > 流水线” |

#### 5. 结果

| 区域 | 内容 |
| --- | --- |
| 标题区 | 查看结果 |
| 主行动区 | 输入任务 ID、查询结果 |
| 辅助区 | 导出摘要 |
| 结果区 | 任务摘要、结果卡片、截图 |
| 帮助区 | 无结果时提示回到“执行”或“更多 > 任务监控” |

#### 6. 更多

| 区域 | 内容 |
| --- | --- |
| 标题区 | 高级与运维 |

| 辅助区 | 角色说明 |
| 结果区 | 无 |
| 帮助区 | 提示普通用户只需使用前 5 个一级入口 |

## C. 设计系统

### C1. Notion 风格基础变量

色板：

- 背景：`#f7f6f3`
- 次级背景：`#f1efe8`
- 卡片：`#ffffff`
- 边框：`#e7e2d8`
- 主文字：`#1f1f1c`
- 次文字：`#6b6a63`
- 主色：`#2f6fed`
- 主色浅底：`#eef4ff`
- 成功：`#2f8f63`
- 危险：`#c85a54`
- 警告：`#b7791f`

字体：

- 中文优先：`"Noto Sans SC", "PingFang SC", "Microsoft YaHei", sans-serif`
- 英文与数字：跟随正文，不再单独强调展示字体

间距：

- `4 / 8 / 12 / 16 / 24 / 32 / 48`

圆角：

- 卡片 `16px`
- 输入框 `12px`
- 小标签 `999px`

阴影：

- 默认卡片：`0 1px 2px rgba(16, 24, 40, 0.04)`
- 浮层：`0 8px 24px rgba(16, 24, 40, 0.08)`

### C2. 组件规范

Button：

- 主按钮只有 1 个，蓝底白字
- 次按钮最多 2 个，白底描边
- 危险按钮只在删除类动作出现

Input / Select / Textarea：

- 高度统一
- 标签在上，错误在下
- 必填显示 `*`

Table / List：

- 以卡片列表优先，表格只用于审计或密集数据
- 列表项只保留关键字段和一个操作入口

Badge：

- 只表达状态，不承载解释

Toast：

- 格式统一：`动作结果 + 下一步`
- 例：`任务创建成功，可直接查看结果`

Modal / Drawer：

- 只放详情，不放主流程

EmptyState：

- 必须包含“当前没有什么” + “下一步去哪里”

Step：

- 用 3 步卡表达主流程
- 每步只保留标题、说明、一个动作

### C3. 文案规范

语气：

- 直接、克制、面向动作
- 少解释系统，多提示下一步

按钮命名：

- `动词 + 对象`
- 例：`上传资产`、`创建任务`、`查看结果`、`提交模型`

错误提示：

- 格式：`动作失败：原因。下一步建议`
- 例：`创建任务失败：缺少资产 ID。请先上传资产或粘贴已有 ID。`

## D. 实施计划（分 PR）

- PR1：只做信息架构与首屏骨架，不动接口
- PR2：统一色板、字体、按钮、输入框、列表与空状态
- PR3：补齐首次使用引导、加载、错误、无权限提示
- PR4：做键盘操作、对比度、响应式和性能收尾

## E. 验收清单

- 首次使用 30 秒内能完成：上传资产 -> 创建任务 -> 查看结果
- 任何页面主按钮只有 1 个，次按钮不超过 2 个
- 空数据、无权限、接口失败都有清晰提示和下一步建议
- 表单有默认值、必填标识和字段级错误反馈
- 全站间距、字号、状态颜色与按钮命名保持统一
