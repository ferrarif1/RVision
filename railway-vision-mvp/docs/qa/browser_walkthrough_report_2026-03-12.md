## 2. 已确认收口的设计问题

### 2.1 多功能整页堆叠

已收口：
- 高复杂页面已统一改成 `页内导航 / 二级导航 / 技术详情后置`
- 默认首屏只显示当前最该做的一步，不再把表单、列表、工作台和说明整页平铺

已覆盖：
- 任务中心：`快速识别 / 创建任务 / 可选模型 / 任务列表`
- 结果中心：`查询结果 / 下一步动作 / 变成训练数据 / 结果列表`
- 模型中心：`模型总览 / 提交与训练协作 / 审批与发布`
- 训练中心：`训练总览 / 创建训练 / 训练机器`
- 流水线中心：`流水线总览 / 注册配置 / 发布管理`

### 2.2 默认视图工程味过重

已收口：
- 默认可见区高频术语已改成业务表达：
  - `任务编号 / 资产编号 / 模型编号 / 流水线编号`
  - `训练机器 / 本机训练机器 / 受控训练机`
  - `最终文本 / 定位框 / 识别引擎`
  - `验证门禁 / 最佳检查点 / 损失曲线 / 准确率曲线`
- 原始 `task_id / asset_id / model_id / pipeline_id / worker / bbox / engine` 主要退到 `技术详情`

### 2.3 长标题、长编号、长版本号溢出

已收口：
- 卡片、工作台摘要、表格单元格、详情展开区、Hero 动作区统一补了：
  - `min-width: 0`
  - `overflow-wrap: anywhere`
  - `word-break: break-word`
- 最近资产、任务卡、模型卡、结果卡、工作台概览、详情 JSON 区已不再因长文本撑破布局

### 2.4 回跳链路不清晰

已收口：
- 任务 -> 结果 -> 返回任务中心
- 训练 -> 验证新模型 -> 结果
- 结果 -> 训练中心 / 任务详情 / 任务中心
- 模型 / 流水线 / 训练页的工作台动作会自动切到对应分区

## 3. 本轮补做的细部修正

- 模型发布工作台：
  - `tenant_code / delivery_mode / authorization_mode / local_key_label / api_access_key_label`
    改为更自然的中文表单语言
- 流水线发布工作台：
  - `tenant_code` 改为 `租户编码`
  - `threshold_version / name` 在技术详情里改为中文
  - `thresholds` 说明改为 `当前阈值规则版本`
- 训练中心数据集版本：
  - 搜索框提示改为 `搜索数据集标签 / 版本说明 / 来源`
- 结果中心回灌训练数据：
  - `task` 改为 `任务`
  - `dataset_label` 错误提示改为 `数据集标签`
  - 用途区任务类型 badge 改成用户可理解的任务中文名
  - 导出成功反馈改成 `数据集资产编号`

## 4. 当前默认展示原则

所有高频控制台页统一遵循：
- 第一眼先看结论、状态和下一步动作
- 第二层再看列表和筛选
- 原始编号、原始 JSON、长哈希、底层枚举统一后置到 `技术详情`

- 极窄窗口下个别 badge 会较早换行，但不再破版
