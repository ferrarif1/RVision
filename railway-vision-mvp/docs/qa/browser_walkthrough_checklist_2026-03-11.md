# Browser Walkthrough Checklist 2026-03-11

目的：把“页面看起来顺不顺、交互会不会乱、长文本会不会撑坏布局”从口头要求变成可重复执行的 QA 清单。

执行原则：
- 每次只走真实主链，不用 synthetic 测试路径代替。
- 每页至少检查桌面宽度下的默认状态、展开高级区、长文本状态、返回链路。
- 发现问题时优先记录“触发动作 -> 实际现象 -> 期望行为”。

## 1. 全站公共项

- 顶部导航切换页面后，标题、摘要、主动作按钮不应换行错位。
- 长 `asset_id / model_id / pipeline_id / task_id` 不应把卡片撑破。
- `details` 展开后不应把相邻卡片高度异常拉伸。
- 右侧信息栏和左侧主表单在内容变多时仍应保持对齐，不出现跨列挤压。
- 返回链路要保留上下文：从结果页返回任务页、从训练页跳模型页、从任务页去结果页后再回退。

## 2. 任务中心

- 快速识别：明确输入 `车号` 后，应直接出现模型选项，不强制先预检。
- 快速识别：选择模型后，“带入下方精确任务”应正确预填模型、任务类型、设备和意图。
- 批量快速识别结果：默认只展开第一条，其余显示摘要；展开/收起后布局不应乱。
- 精确创建任务：高级控制折叠前后，表单宽度不应跳变。
- 任务列表：长文件名、长任务 ID、长模型版本号不应撑坏卡片。
- 任务详情：默认先看结论，技术详情在 `Advanced` 内。

## 3. 结果中心

- OCR 结果顶部概览应优先展示 `车号结果 / 稳定文本 / 待复核 / 平均执行耗时`。
- 不应再出现 `car_number · 1` 这类内部标签直出。
- 结果卡默认不应先暴露 `result_id / model_id / stage / INFO`。
- “下一步动作”工作台应根据结果状态给出明确建议。
- “结果回灌工作台”导出后，应能清楚看到下一步去训练中心。

## 4. 训练中心

- 训练工作台概览要跟当前焦点作业联动。
- 创建训练作业：默认预设应可直接用，高级 JSON 折叠前后布局稳定。
- 训练作业列表：卡片默认先看状态、候选模型、执行节点，不先暴露长 ID。
- Worker 区：默认只看活跃机器，历史异常不应压住主视图。
- 训练摘要：候选模型验证入口和训练曲线要在一屏内看清，不互相挤压。

## 5. 模型中心

- 模型工作台概览要根据当前模型状态切换为评估 / 审批 / 发布入口。
- 模型列表卡片应先看状态、风险、来源，不先看 hash。
- 审批工作台：建议样本、最近验证结果、审批说明三块在同屏内层级清晰。
- 发布工作台：设备、买家、交付方式在桌面宽度下不应换行混乱。

## 6. 流水线中心

- 流水线工作台概览要给出明确发布建议。
- 注册表单默认只暴露高频字段，`expert_map / thresholds / fusion_rules / config` 在高级区。
- 流水线列表卡片默认先看状态、阈值版本、路由模型摘要。
- 发布工作台里的设备、买家、流量比例在内容变化后不应挤压失衡。

## 7. 资产 / 设备 / 审计 / 设置

- 这些页的首屏 `page hero` 高度和按钮行要保持一致。
- 资产和设备列表里的长文件名、长设备编码应自动换行或截断。
- 审计中心和设置页默认先看业务结论，不先看长 JSON。

## 8. 当前优先修复顺序

1. 任务中心与结果中心的上下文回跳和批量结果展开。
2. 训练中心卡片、Worker 区、训练摘要的布局稳定性。
3. 模型中心与流水线中心工作台表单的长文本与折叠区。
4. 次级页面的长文本、空状态和响应式细节。
