# CEO Track

## Mission

CEO 轨道负责定义这个平台为什么值得做、怎么赚钱、靠什么形成控制力，以及商业承诺与交付能力之间是否一致。

## Business Truth To Keep Consistent

- 平台的核心控制点是数据主权、模型发布权、交付权、收费权。
- 供应商提供算法能力，但不直接拿走客户数据、最终成果模型和发布控制权。
- 当前 MVP 已经具备“模型注册、审批发布、边缘推理、结果回传、审计留痕”的交易与交付底座。
- 当前并未完成真正的受控训练平台，因此对外叙事不能承诺“现成的分布式训练 SaaS”，只能承诺“受控交付平台 + 训练控制面演进路线”。

## Executive-Level Outputs

| Artifact | Purpose |
|---|---|
| PR / FAQ | 对外讲清客户价值、为什么是现在、为什么是我们 |
| Pricing Policy | 定义授权方式、计费口径、结算规则 |
| Partner Policy | 定义供应商合作边界、收益分配、交付义务 |
| KPI Deck | 跟踪收入、交付、可用性、续费、模型活跃度 |
| Phase Report | 对齐阶段结果、风险与下一阶段投入 |

## Decision Questions

每次经营层决策至少要回答：

- 平台到底卖什么：单模型、流水线、交付能力，还是持续运营能力。
- 本阶段承诺的能力是否真的已经可交付。
- 是否有足够证据支持定价、续费与合作伙伴条款。
- 哪些能力是需要自建，哪些适合合作或外采。

## Review Questions

- 对外销售口径是否和产品、技术现状完全一致。
- 是否把目标态包装成现状能力。
- 商业模式是否强化了平台控制权，而不是被供应商反向控制。
