# 智能引导 / LLM 工作台（2026-03-14）

- Owner: Engineering / Product
- Status: Implemented
- Last Updated: 2026-03-14 13:05 CST
- Scope: 为“上传图片 + 说清目标 + 选择 LLM 模式 + 获得下一步动作”提供正式入口，避免用户在任务、训练、模型、审批之间自己猜流程。

## 1. 为什么要做

当前平台已经具备：

- 资产上传
- 任务创建与验证
- 训练与待验证模型回收
- 审批 / 发布 / 审计

但对普通用户来说，仍存在一个问题：

- 我上传了图片，也知道自己想识别什么
- 但我不一定知道下一步应该去任务中心、训练中心、模型中心还是审批中心

智能引导工作台就是用来解决这个入口层问题。

## 2. 当前已落地能力

### 2.1 页面入口

- 路由：`/assistant`
- 顶部导航名称：`智能引导`
- 页面形态已重构成：
  - 对话优先的主舞台
  - 单独的右侧上下文抽屉
  - 底部统一输入区
  - 不再把目标输入、模型库、下载任务平铺在同一屏
- 全局主题已新增：
  - `夜幕金`
  - `ChatGPT 清透白`
  - `夏日奶油色`
  智能引导页内切换会同步作用于全站，不再只限当前页。

### 2.2 两种大模型模式

- `API 模式`
  - 支持 OpenAI-compatible 接口
  - 兼容 SiliconFlow / DashScope 这类兼容端点
  - 用户可临时填写：
    - `base_url`
    - `api_key`
    - `model_name`
- `本地模型模式`
  - 平台内置精选前 10 个开源模型
  - 不跳转第三方 UI
  - 可直接发起下载
  - 可查看下载任务和取消任务

### 2.4 智能下一步引导

输入可来自：

- 目标描述
- 显式任务类型
- 显式模型编号
- 已上传资产编号
- 当场上传的图片

系统当前会综合：

- 目标文本关键词
- 已有资产
- 现有可用模型
- 当前任务族状态
- 待验证 / 已审批 / 已发布模型状态

输出：

- 推断任务类型
- 当前状态判断
- 主推荐动作
- 次推荐动作
- 可直接跳转的页面入口
- 可直接写入的预填参数

### 2.5 人工控制优先，自动调整可选

这条产品线的硬约束是：

- LLM 负责建议、预填、导航
- 自动调整只是可选项
- 不能取代人工对模型、代码、参数的直接控制

所以后续实现必须满足：

- 保留手动选模型
- 保留手动改训练参数
- 保留手动改代码 / 算法配置入口
- 自动生成或自动调整的内容必须：
  - 明确标注来源
  - 可撤销
  - 可覆盖
  - 可在提交前继续手动编辑

当前已落地的默认行为：

- 智能引导默认是 `仅导航，不改字段`
- 只有用户显式切到 `跳转并带建议过去`，系统才会写入可编辑预填
- 任务页、训练页、模型页会显示来源条，并提供：
  - 清空建议
  - 返回智能引导
- 智能引导页内的主题切换不会影响这条原则：
  - 视觉可以更像 ChatGPT
  - 但交互上仍然默认“人工控制优先”

## 3. 当前已覆盖的典型场景

### 3.1 用户上传图片，想知道该不该直接验证现有模型

系统会优先判断：

- 当前任务族是否已有已发布模型
- 是否存在待验证模型
- 当前更适合：
  - 直接在任务中心验证
  - 先进入训练数据准备
  - 先进入审批工作台

### 3.2 用户明确知道任务类型，但不知道该去哪里

例如：

- `inspection_mark_ocr`
- `door_lock_state_detect`

系统会直接给出：

- 去任务中心验证
- 去训练中心准备数据
- 去模型中心看待验证模型

### 3.3 用户想先选一个本地大模型辅助判断

系统支持：

- 选本地模型
- 查看本地模型基础信息
- 直接发起下载
- 在不离开平台的前提下继续做规划

## 4. 当前不声称已完成的部分

以下内容还不是当前已落地能力：

- 本地大模型推理服务一键启动
- 平台内真正执行多轮视觉理解对话
- 自动把 LLM 规划结果直接编排成跨页面工作流代理
- 本地模型存储配额 / 淘汰策略 / 节点资源调度

所以当前这项能力的定位是：

- 已落地：入口、模型目录、下载任务、智能规划、跳转引导
- 未落地：完整 Agent 化工作流执行器

## 5. 当前验证证据

- `GET /assistant/provider-modes -> 200`
- `GET /assistant/local-models -> 200`
- `GET /assistant/local-models/download-jobs -> 200`
- `POST /assistant/plan -> 200`
- `POST /assistant/local-models/download -> 200`
- `POST /assistant/local-models/download-jobs/{job_id}/cancel -> 200`

已在运行环境中验证：

- 模式返回包含 `api / local`
- 本地模型目录返回 `10` 条精选模型
- 目标为“定检标记识别”时，可推断到 `inspection_mark_ocr`
- 可返回主推荐动作和次推荐动作
- 本地模型下载任务可创建并取消

## 6. 下一步建议

如果继续扩这条线，优先级建议是：

1. 把“手动控制优先、自动调整可选”落实到训练 / 模型 / 流水线 / 任务承接页
2. 把智能引导接到更强的跨页状态机
3. 增加本地模型存储治理与配额策略
4. 给 API 模式加更正式的密钥托管与脱敏策略
5. 再决定是否接入真正的本地 LLM 推理执行器
