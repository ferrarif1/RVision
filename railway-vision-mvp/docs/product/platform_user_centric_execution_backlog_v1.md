## 1. 执行策略

- 先做主路径，再做高级能力。
- 每个条目都有“交付物 + 验收标准 + 责任人”。
- 每周评审一次状态（Done / In Progress / Risk / Blocked）。

## 2. Week 1-2（当前执行窗口）

## 2.1 IA 与导航重组

- [ ] 输出角色导航草案（买家/供应商/平台）
- [ ] 定义一级导航最终版本与迁移策略
- [ ] 输出旧入口到新入口映射表

交付物：

- IA 图（mermaid 或 Figma 链接）
- 路由迁移表

验收标准：

- 任意页面可在 2 次点击内回到角色工作台

## 2.2 角色工作台

- [ ] 买家工作台卡片（上传 -> 执行 -> 结果）
- [ ] 供应商工作台卡片（提交 -> 训练协作 -> 候选）
- [ ] 平台工作台卡片（审批 -> 发布 -> 审计）

交付物：

- 页面线框 + 文案规范

验收标准：

- 新用户 30 秒内可解释“下一步做什么”

## 2.3 状态统一

- [ ] 全站状态矩阵（loading/empty/error/permission/stale）
- [ ] 错误提示模板统一（动作失败 + 原因 + 下一步）

交付物：

- 状态组件清单
- 文案字典

验收标准：

- 关键流程无“裸错误”与无动作空白页

## 3. Week 3-5（训练中心）

## 3.1 训练作业可视化

- [ ] 作业列表：状态、耗时、失败原因、候选模型关联
- [ ] Worker 列表：健康、资源、待处理作业数
- [ ] 作业详情：阶段进度 + 指标 + 输出摘要

验收标准：

- 训练失败后，用户可在 1 分钟内定位原因并重试

## 3.2 训练协议与执行

- [ ] 统一训练输入/输出契约（含占位变量、指标文件）
- [ ] 外部训练命令接入规范
- [ ] 失败分类（可重试/不可重试）

验收标准：

- 100% 训练作业具备结构化 output_summary

## 4. Week 6-8（验证与审批）

- [x] 候选模型自动验证门禁（验证资产 / 指标 / best checkpoint / runtime 契约）
- [ ] 审批编排（通过/拒绝/补充材料）
- [x] 发布前风险摘要卡片

验收标准：

- 无验证报告的候选模型无法进入发布

## 5. Week 9-12（商用化）

- [ ] 授权范围管理（租户/设备/版本）
- [ ] 调用计量与结算台账
- [ ] 合规证据包（审计 + 发布 + 验证）导出

验收标准：

- 可生成单租户端到端可审计交付报告

## 8. 当前工程待办（2026-03-08）

已记录，后续按优先级继续推进：

- [x] 训练/验证资产支持 0-n 个 asset_id；允许空列表创建训练作业
- [x] 训练/验证资产支持 ZIP 数据集包；ZIP 内支持多层文件夹与多图片/多视频资源
- [x] 快速识别结果支持误检删除、手工补框、图上拖拽修订
- [x] 快速识别导出数据集资产时自动生成版本记录，并进入训练数据集版本流
- [x] 训练作业支持取消 / 重试 / 改派到指定 worker host
- [x] 运行中取消支持 worker 侧停止协议
- [x] worker 独立部署物料已收口到 `deploy/training-worker/`，并支持构建独立 bundle
- [x] 后端 API 自动化回归：`auth / assets / training / edge / tasks / results`
- [x] 训练 Worker 外部训练命令契约：失败分类、退出码归一、指标历史 schema 固化
- [x] 训练页真实曲线：epoch 级 metrics history、best checkpoint、收敛趋势
- [x] 数据集版本高级操作：推荐验证集、回滚到某版、版本间样本级差异筛选
- [x] 训练作业超时回收与告警：worker 失联、长时间 RUNNING、产物上传超时
- [ ] 数据保留策略：训练产物、上传临时文件、历史 QA 报告自动清理
- [ ] schema 快照自动导出，替代当前 `snapshot-at-tip + --write` 人工同步

## 9. 下一阶段执行顺序（建议）

按当前收益和风险排序，建议继续按下面顺序推进：

1. schema 快照自动导出，减少手工同步动作。
2. 数据保留策略自动化，避免运行期产物持续堆积。
3. 审批编排（通过 / 拒绝 / 补充材料）和发布前证据包导出。
4. 更细粒度的训练资源调度与容量治理。
