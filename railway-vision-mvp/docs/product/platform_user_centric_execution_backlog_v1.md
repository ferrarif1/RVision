# 平台用户视角执行清单（V1）

- Owner: Product + Engineering + QA
- Status: In Execution
- Last Updated: 2026-03-05
- Parent: `platform_user_centric_implementation_v1.md`

## 1. 执行策略

- 先做主路径，再做高级能力。
- 每个条目都有“交付物 + 验收标准 + 责任人”。
- 每周评审一次状态（Done / In Progress / Risk / Blocked）。

## 2. Week 1-2（当前执行窗口）

## 2.1 IA 与导航重组

- [ ] 输出角色导航草案（买家/供应商/平台）
- [ ] 定义一级导航最终版本与迁移策略
- [ ] 输出旧入口到新入口映射表

交付物：

- IA 图（mermaid 或 Figma 链接）
- 路由迁移表

验收标准：

- 任意页面可在 2 次点击内回到角色工作台

## 2.2 角色工作台

- [ ] 买家工作台卡片（上传 -> 执行 -> 结果）
- [ ] 供应商工作台卡片（提交 -> 训练协作 -> 候选）
- [ ] 平台工作台卡片（审批 -> 发布 -> 审计）

交付物：

- 页面线框 + 文案规范

验收标准：

- 新用户 30 秒内可解释“下一步做什么”

## 2.3 状态统一

- [ ] 全站状态矩阵（loading/empty/error/permission/stale）
- [ ] 错误提示模板统一（动作失败 + 原因 + 下一步）

交付物：

- 状态组件清单
- 文案字典

验收标准：

- 关键流程无“裸错误”与无动作空白页

## 3. Week 3-5（训练中心）

## 3.1 训练作业可视化

- [ ] 作业列表：状态、耗时、失败原因、候选模型关联
- [ ] Worker 列表：健康、资源、待处理作业数
- [ ] 作业详情：阶段进度 + 指标 + 输出摘要

验收标准：

- 训练失败后，用户可在 1 分钟内定位原因并重试

## 3.2 训练协议与执行

- [ ] 统一训练输入/输出契约（含占位变量、指标文件）
- [ ] 外部训练命令接入规范
- [ ] 失败分类（可重试/不可重试）

验收标准：

- 100% 训练作业具备结构化 output_summary

## 4. Week 6-8（验证与审批）

- [ ] 候选模型自动验证门禁（精度、鲁棒、性能）
- [ ] 审批编排（通过/拒绝/补充材料）
- [ ] 发布前风险摘要卡片

验收标准：

- 无验证报告的候选模型无法进入发布

## 5. Week 9-12（商用化）

- [ ] 授权范围管理（租户/设备/版本）
- [ ] 调用计量与结算台账
- [ ] 合规证据包（审计 + 发布 + 验证）导出

验收标准：

- 可生成单租户端到端可审计交付报告

## 6. 每周例会模板

- 本周完成（Done）
- 下周计划（Plan）
- 风险与阻塞（Risk/Blocked）
- 需要决策（Decision Needed）
- 指标变化（KPI Trend）
