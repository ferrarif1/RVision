# 平台用户视角完整实现方案（V1）

- Owner: Product Track
- Status: Proposed -> In Execution
- Last Updated: 2026-03-05
- Scope: 平台控制台目标态产品实现（不受当前代码限制）
- Non-goals: 不定义具体技术栈、不约束单个接口实现细节

## 1. 目标定义（一句话）

把 VisionHub 从“可演示的能力集合”升级为“可运营、可审计、可商业化交付的平台产品”：

- 买家：明确上传-执行-回查闭环
- 供应商：明确提交-训练协作-候选交付闭环
- 平台：明确审批-发布-授权-结算闭环

## 2. 产品原则（用户第一）

1. 先任务后模块：页面按“用户要完成什么”组织，而不是按系统模块组织。
2. 默认路径优先：每个角色 1 条默认路径 + 少量高级入口。
3. 治理即体验：权限、审计、主权边界以“系统默认行为”体现，不让用户额外学习。
4. 可运营可量化：每条主路径可观测、可度量、可复盘。

## 3. 用户与核心任务

### 3.1 买家（operator / auditor）

- 核心任务：上传资产并获得可追溯结果
- 成功定义：3 步完成闭环（上传 -> 执行 -> 结果）
- 风险点：找不到下一步、结果不可追责、导出无审批

### 3.2 供应商（supplier engineer）

- 核心任务：提交算法能力并交付可验收候选模型
- 成功定义：能看到当前状态、验收要求、下一步动作
- 风险点：提交后黑盒、训练反馈不足、交付口径不统一

### 3.3 平台管理员（platform admin/operator/auditor）

- 核心任务：治理模型生命周期并保障合规交付
- 成功定义：审批、发布、授权、审计与结算在一条连续路径中
- 风险点：治理动作分散、证据链不完整、回滚路径不清晰

## 4. 目标信息架构（IA）

一级导航建议：

1. 工作台（按角色）
2. 资产中心
3. 训练中心
4. 模型中心
5. 流水线中心
6. 运行中心（任务/设备/结果）
7. 审计与结算

说明：

- “训练中心”单独升格，避免训练能力长期隐藏在文档和脚本层。
- “运行中心”统一任务、设备、结果，减少用户跨页跳转成本。

## 5. 关键流程设计（目标态）

## 5.1 买家推理闭环（必须默认可用）

1. 上传资产（含用途、数据等级）
2. 选择流水线并创建任务
3. 查看结果（结构化 + 截图 + 模型指纹）
4. 需要时发起导出（受审批策略约束）

验收：首次用户 30 秒内可理解路径，3 分钟内完成首个成功任务。

## 5.2 供应商交付闭环（可协作、可追踪）

1. 提交初始算法/基线模型
2. 进入训练作业协作（查看作业状态、训练指标、失败原因）
3. 上传候选模型包（标准 manifest/hash/signature）
4. 跟踪审批结果与修订意见

验收：候选模型交付后，供应商可定位“当前在哪一关、谁在处理、还缺什么”。

## 5.3 平台治理闭环（主权边界可执行）

1. 审批候选模型（测试证据 + 风险摘要）
2. 发布模型（授权范围：租户/设备/版本）
3. 运行监控（效果、稳定性、告警）
4. 审计与结算（操作追溯、授权计量）

验收：任意线上结果可追溯到发布记录与模型指纹。

## 6. 训练中心目标能力（从“控制面”到“产品面”）

必须补齐四层：

1. 作业层：创建、分配、状态机、失败重试
2. 执行层：worker 真实训练协议、可注入训练命令
3. 观测层：日志、指标、阶段进度、错误模板
4. 治理层：自动验证门禁、审批晋级、回滚策略

说明：当前仓库已有控制面骨架与最小执行脚本，目标态需补足自动验证与容量治理。

## 7. 页面级“完成标准”

每个关键页必须同时具备：

- 唯一主 CTA（次要 CTA <= 2）
- Loading / Empty / Error / Permission denied / Stale 状态
- 成功后下一步推荐
- 失败后可重试且能定位
- 审计相关操作有留痕提示

## 8. 版本化落地计划（12 周）

### Phase A（W1-W2）：认知清晰化

- 角色工作台改造
- 一级导航按任务重组
- 全站状态组件与文案统一

### Phase B（W3-W5）：训练执行可运营

- 训练中心可视化（作业、worker、指标）
- 失败分类与重试策略
- 训练执行证据标准化

### Phase C（W6-W8）：验证与审批自动化

- 自动验证门禁
- 审批编排（人机协同）
- 发布前风险摘要

### Phase D（W9-W12）：商用化闭环

- 授权范围管理
- 调用计量与结算
- 合规证据包导出

## 9. KPI 与守护指标

北极星指标：

- 首次关键任务完成率（按角色）
- 发布模型可追溯率（目标 100%）

守护指标：

- 训练作业失败率/平均恢复时间
- 结果回传延迟
- 导出审批违规拦截率
- 审计缺失率（目标 0）

## 10. 依赖与风险

依赖：

- 平台/算法/数据/安全/运维跨团队协同
- 统一术语与事件口径

风险：

- 把“目标态”误写成“已实现”
- 先做复杂能力导致主路径迟迟不可用
- 合规策略未产品化，落地后靠人工兜底

## 11. 发布门禁（Go/No-Go）

- 三类角色主路径全通过
- 审计关键动作覆盖完整
- 训练中心关键状态可观测
- 灰度发布与回滚演练通过
- 文档与实现状态一致
