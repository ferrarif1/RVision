# 平台收费模型与分账规则（通用版）

> 适用场景：客户上传数据资产，供应商提供初始算法与可选预训练模型，平台受控微调、验证审批并授权交付可用模型。

## 0. 统一业务主线

- 客户用户上传图片或视频资产，资产可用于训练、微调、测试验收或推理。
- 供应商上传初始算法与可选预训练模型，在平台受控环境内结合客户数据反复微调，形成候选成果模型并提交审批。
- 平台管理员结合客户测试数据验证模型有效性，审批并发布模型。
- 授权客户设备通过模型 API 或授权密钥使用加密模型，本地运行时完成解密。

## 1. 收费对象与角色

- 平台方（甲方）：交易撮合、托管训练、验收发布、商业分发与结算。
- 供应商（乙方）：初始算法/初始模型提供、微调支持、技术支持。
- 买家（丙方）：提交识别需求、提供数据、采购授权并使用模型服务。

## 2. 费用构成（建议按模块拆分）

## 2.1 一次性交付类

- 需求分析费：需求澄清、指标定义、实施方案。
- 数据治理费：数据清洗、脱敏、标注、质检。
- 训练执行费：托管训练资源消耗（GPU/CPU/存储）。
- 验收费：精度验收、稳定性验收、合规验收。
- 发布交付费：签名发布、灰度上线、验收归档。

## 2.2 持续服务类

- 订阅费：按月/季度收取平台使用费用。
- 调用量费用：按 API 次数、推理帧数或处理时长计费。
- SLA保障费：可用性、响应时延、故障响应保障。
- 迭代优化费：难例回流、复训、版本升级。

## 3. 计费口径（可落地字段）

- 训练资源：`gpu_hours`、`cpu_hours`、`storage_gb_day`
- 数据治理：`label_count`、`qa_rounds`
- 验收执行：`test_rounds`、`scene_coverage`
- 运行服务：`api_calls`、`video_minutes`、`active_devices`
- 支持服务：`support_tickets`、`response_sla_level`

## 4. 参考定价公式

总费用（项目期）可按以下方式计算：

```text
Total = 一次性交付费用 + 持续服务费用

一次性交付费用
= 需求分析费 + 数据治理费 + 训练执行费 + 验收费 + 发布交付费

持续服务费用（按周期）
= 订阅费 + 调用量费用 + SLA保障费 + 迭代优化费
```

调用量费用示例：

```text
调用量费用 = (API调用次数 × 单价) + (视频分钟数 × 单价) + (在线设备数 × 单价)
```

## 5. 分账模型（平台与供应商）

## 5.1 建议分账范围

- 平台保留收入：平台服务费、验收发布费、密钥/审计/合规费、商业分发费。
- 供应商分成收入：算法服务费、优化支持费、专项技术服务费。

## 5.2 分账计算示例

```text
可分账基数 = 买家实付金额 - 税费 - 通道费 - 平台固定服务费
供应商应得 = 可分账基数 × 供应商分成比例
平台应得 = 买家实付金额 - 供应商应得 - 税费 - 通道费
```

说明：

- 分成比例按供应商等级、算法效果、SLA表现分档。
- 若供应商未达验收指标，可触发降档或不分账。

## 6. 结算与对账流程

1. 买家下单（需求+范围+验收指标）。
2. 平台托管训练并完成验收。
3. 验收通过后生成交付单与结算单。
4. 平台按账期（T+7 / T+30）与供应商结算。
5. 支持退款/赔付场景下执行冲正与追溯。

## 7. 商业与风险控制条款

- 平台独占对外分发与收费接口，供应商不得绕过平台直接收费。
- 平台保留模型发布、下架、回滚与停服权。
- 数据与微调后模型归平台控制，供应商不得对外复用。
- 验收失败、SLA违约、合规违规应触发扣款或暂停合作。

## 8. 建议账单结构（可用于系统化）

- 订单主单：`order_id`、买家、供应商、项目范围、签约金额。
- 资源账单：训练资源、调用量、存储量、支持工单。
- 验收账单：验收轮次、通过状态、失败原因。
- 分账账单：分账基数、比例、应付金额、实付金额。
- 审计账单：审批人、审批时间、版本号、发布记录。

## 9. 对外口径（销售/法务统一）

- 平台模式：算法交易 + 托管训练 + 验收交付 + 分账结算。
- 权责边界：供应商提供能力，平台控制数据、模型、发布与商业分发。
- 收费逻辑：按“交付价值 + 资源消耗 + 服务保障”三层收费。
